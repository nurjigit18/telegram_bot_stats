# OpenAI Integration Setup Guide

## Обзор
Бот теперь поддерживает интеграцию с OpenAI для обработки естественного языка при сохранении данных о товарах. Пользователи могут вводить информацию в свободной форме, а ИИ автоматически извлечет необходимые поля.

## Настройка

### 1. Установка зависимостей
```bash
pip install openai
```

### 2. Настройка переменных окружения
Добавьте в ваш `.env` файл:
```env
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-3.5-turbo
```

### 3. Опциональные настройки
- `OPENAI_MODEL` - модель OpenAI (по умолчанию: gpt-3.5-turbo)
- Если `OPENAI_API_KEY` не установлен, бот будет работать только со строгим форматом

## Новые возможности

### 1. Ввод в естественном языке
Пользователи могут описывать товары естественным языком:

**Примеры:**
```
Красные рубашки, 100 штук, Казань: 50S, 25M, 25L. 12/12/2025 - 15/12/2025.

Мне нужно сохранить синие джинсы. У нас 80 штук: склад Москва M-40 L-40. Отправляем 20 января, прибудут 25 января.

Белые футболки 50 штук, отправка завтра, прибытие через неделю.
```

### 2. Дружелюбные сообщения об ошибках
Вместо технических сообщений об ошибках, ИИ генерирует дружелюбные запросы:

**Было:**
```
❌ Найдены следующие ошибки:
• Неверное количество полей
```

**Стало:**
```
🤔 Я понял, что речь идёт о красных рубашках, но не смог определить точное количество и даты. Не могли бы вы подсказать сколько штук и когда планируете отправку?
```

### 3. Частичное извлечение данных
ИИ может извлечь частичную информацию и попросить недостающие данные:
- Показывает что уже понял
- Просит конкретную недостающую информацию
- Использует тёплый, дружелюбный тон

## Логика работы

### 1. Приоритет обработки
1. **OpenAI парсинг** (если включен) - первый приоритет
2. **Строгий формат** - резервный вариант
3. **Дружелюбные сообщения** - для недостающих данных

### 2. Обработка ошибок
- Если OpenAI API недоступен → автоматический переход к строгому формату
- Если данные неполные → генерация дружелюбного запроса
- Если валидация не прошла → ИИ объясняет что нужно исправить

### 3. Подтверждение данных
Бот показывает что извлёк ИИ перед сохранением:
```
🤖 ИИ извлек следующие данные:
📦 Название изделия: рубашки
🎨 Цвет изделия: красные
📊 Количество: 100 шт
🏪 Склады и размеры: Казань: S-50 M-25 L-25
📅 Дата отправки: 12/12/2025
📅 Дата прибытия: 15/12/2025
```

## Тестирование

### Запуск тестов
```bash
python test_openai_integration.py
```

### Тестовые сценарии
1. **Полные данные** - проверка успешного извлечения
2. **Частичные данные** - тестирование дружелюбных запросов
3. **Неполные данные** - проверка обработки ошибок
4. **Смешанные форматы** - русский/английский язык

## Мониторинг и логирование

### Логи OpenAI
- Успешные запросы логируются как INFO
- Ошибки API логируются как ERROR
- Частичные данные логируются как WARNING

### Отслеживание использования
- Все вызовы OpenAI API логируются
- Можно отслеживать стоимость через логи
- Таймауты настроены на 30 секунд

## Безопасность

### Обработка ошибок
- API ключ не логируется
- Graceful fallback при недоступности OpenAI
- Валидация всех извлечённых данных

### Ограничения
- Максимум 1000 токенов на ответ
- Таймаут 30 секунд
- Низкая температура (0.1) для консистентности

## Отключение OpenAI

Если нужно отключить OpenAI интеграцию:
1. Удалите `OPENAI_API_KEY` из `.env`
2. Или установите `OPENAI_API_KEY=""`
3. Бот автоматически переключится на строгий формат

## Поддержка

При возникновении проблем:
1. Проверьте логи в `logs/bot.log`
2. Убедитесь что API ключ корректный
3. Проверьте баланс OpenAI аккаунта
4. Запустите тестовый скрипт для диагностики
